设计出发点

不是一次性生成

探究式教案需要从现象、问题、猜想、实验到结论逐步形成，系统因此以阶段为组织单位。

教师始终在环

教师输入、确认阶段草稿、修订内容，模型只提供辅助分析和整合建议。

专家按阶段路由

七个阶段绑定七个专家 Agent，后端根据当前阶段自动选择，不让前端手动切换专家。

过程可恢复

每轮保存教师、专家、主导师三条消息，回滚时恢复上一版草稿，保证协作过程可追溯。

02

系统编排方式

Vue 3 工作台

承载流程选择、双角色对话、阶段草稿编辑和 Markdown 导出。

FastAPI 编排层

负责会话、阶段状态机、Agent 路由、SSE 流式输出、消息落库与回滚。确定性的流程控制都放在 Python 后端里。

阶段专家 + 主导师

阶段专家给出专业意见，主教学导师贯穿全链路，整合历史、草稿和专家输出。

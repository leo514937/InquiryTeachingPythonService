# Day 3 - 对话与草稿

## 当前状态

已完成，并已通过自动化测试。

## 分工

- 主对话：SSE 流式返回、阶段事件、回复落库
- 草稿链路：右侧草稿同步、手动保存、阶段定稿
- 回滚链路：最近一轮对话撤销、阶段回退

## 目标

把人机协同的主对话链路做成 SSE 流式接口，并把草稿实时同步到阶段输出。

## 开发任务

- 实现 `/api/sessions/{session_id}/chat` 的 SSE 流式返回。
- 支持 `stage`、`agent`、`delta`、`draft`、`done` 事件。
- 保存用户消息、导师回复、草稿内容和阶段输出。
- 支持最近一轮回滚，以及阶段回退。

## 产物

- `app/api/chat.py`
- `app/services/draft_service.py`
- `app/api/sessions.py` 中的回滚与草稿接口

## 验收标准

- 发送一条消息后，前端能逐段看到流式回复。
- 草稿可以随着回复实时更新并手动保存。
- 回滚后消息和阶段状态恢复到预期位置。

## 已实现

- 主对话返回 `stage`、`agent`、`delta`、`draft`、`done` 五类 SSE 事件。
- 用户消息、阶段专家回复、主导师回复和阶段草稿均会持久化。
- 新增 `chat_turns` 轮次记录，回滚时按准确的消息 ID 删除一轮问答。
- 每轮正常对话包含三条消息，`chat_turns.expert_message_id` 关联阶段专家回复。
- 回滚最近一轮对话时，阶段草稿会恢复到该轮对话前的内容。
- 阶段回退只调整流程位置并重新开放上一阶段，不误删历史对话。
- 支持右侧草稿手动保存，并保留 Markdown 内容。
